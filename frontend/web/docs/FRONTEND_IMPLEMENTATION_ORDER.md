# Порядок фронтенда и карта маршрутов

Сопоставление [requirements_for_pages.mdc](../../../extra_tools/.cursor/rules/requirements_for_pages.mdc) и текущего приложения в [App.jsx](../src/App.jsx). Итерации — ориентир для развития, а не жёсткий срок.

## Текущие маршруты

| Путь | Компонент | Роль в спеке |
|------|-----------|--------------|
| `/`, `*` | редирект на `/sign-in` | Временная заглушка; в продукте часто делают редирект на `/main` для гостя |
| `/sign-in` | `SignIn` | Вход; восстановление пароля — в спеке, в коде может отсутствовать |
| `/sign-up` | `SignUp` | Регистрация (сейчас одна форма; в спеке — многошаговый мастер) |
| `/main` | `MainPage` | Главная: лента, поиск, фильтры, карта сверху, «Ещё» — частично |
| `/create-room` | `CreateRoomPage` | Создание активности (в спеке — пошагово как Авито + модерация после отправки) |
| `/profile` | `ProfilePage` | ЛК и подстраницы — в спеке шире (уведомления, заявки, «мои активности») |
| `/qa` | `QADataPage` | Q&A список |
| `/qa/new` | `CreateQuestionPage` | Создание вопроса (только при наличии токена) |
| `/cards/default-0` | `DefaultCard0` | Заглушка страницы активности; целевой маршрут вида `/activities/:id` |
| `/questions/default-0` | `DefaultQuestion0` | Заглушка страницы вопроса |

## Итерации (как в плане спецификации)

1. **Каркас** — `App.jsx`, `AppHeader`, тема (moon-иконка без логики), i18n (шар языка в спеке — нет в UI), сессия 30 мин (бэкенд/токены).
2. **Главная** — пагинация «Ещё», карта над лентой, hover-фото на карточках, сортировка/фильтры в данных.
3. **Auth** — мастер регистрации по шагам, код на почту, опциональный второй фактор при входе.
4. **Страница активности** — динамический `:id`, карусель, CTA вступить/заявка, карта, чат, вопрос админам, QR; гостю без доски (реализовано на `DefaultCard0` как пример).
5. **ЛК** — отдельные роуты: уведомления, персональные данные, мои активности, отправленные заявки.
6. **Админ-режим активности** и **мастер создания** + UI статуса модерации.
7. **Q&A** — детальная страница вопроса, расширенная страница активности под ветку вопросов.

## Заметные пробелы относительно спеки

- Нет роутов `/login`/`/register` как канонических (есть редиректы на `/sign-in` и `/sign-up`).
- Нет страниц: восстановление пароля, обучение (слайды), пользовательское соглашение PDF, уведомления, заявки, «мои активности» как отдельные URL.
- Главная: кнопка «Создать активность» скрыта для гостя (ФТ); карта и «Ещё» ещё не сделаны.
